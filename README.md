# 电子科技大学计算机组成与系统结构 II 综合实验

本项目通过 Verilog HDL 语言实现了支持 12 条 MIPS 指令的单周期 CPU 和 5 级流水线 CPU，并包含了对数据冒险和控制冒险的解决方案。

## 项目简介

本实验旨在深入理解 CPU 的工作原理和硬件架构。项目从基础的取指电路设计开始，逐步实现单周期 CPU，最终完成具有冒险处理能力的 5 级流水线 CPU。

* **开发语言：** Verilog HDL
* **开发环境：** Xilinx ISE Design Suite 14.7 集成开发系统
* **目标平台：** 32 位指令集架构

## 支持的指令集

本项目设计的 CPU 支持以下 12 条 32 位指令：

| 指令类别 | 指令 | 功能说明 |
| :--- | :--- | :--- |
| **算术逻辑运算** | `add` | 寄存器加法 (`rd ← rs + rt`) |
| | `sub` | 寄存器减法 (`rd ← rs - rt`) |
| | `subu` | 寄存器无符号减法 (`rd ← rs - rt`) |
| | `slt` | 有符号比较，若 `rs < rt` 则置位 (`rd ← rs < rt ? 1 : 0`) |
| | `sltu` | 无符号比较，若 `rs < rt` 则置位 (`rd ← rs < rt ? 1 : 0`) |
| | `ori` | 立即数按位或 (`rt ← rs \| ZeroExt(imm)`) |
| | `addiu` | 立即数无符号加法 (`rt ← rs + SignExt(imm)`) |
| **数据传输** | `lw` | 取字 (`rt ← memory[rs + SignExt(imm)]`) |
| | `sw` | 存字 (`memory[rs + SignExt(imm)] ← rt`) |
| **控制转移** | `beq` | 条件分支，相等则跳转 (`if(rs == rt) PC ← PC + 4 + SignExt(imm) << 2`) |
| | `j` | 无条件跳转 (`PC ← {PC[31:28], target, 2'b00}`) |

## 架构设计

### 1. 单周期 CPU

单周期 CPU 的特点是在一个时钟周期内执行完一条指令。它包含了取指模块 (`Fetch`)、算术逻辑单元 (`ALU`)、寄存器组 (`Registers`)、数据存储器 (`DataMem`)、控制单元 (`CU`) 等核心部件，通过数据通路连接而成。

### 2. 5 级流水线 CPU

流水线 CPU 将每条指令的执行过程划分为 5 个阶段（取指 IF、译码 ID、执行 EXE、访存 MEM、写回 WB），通过在各个阶段间添加流水线寄存器 (`IF_ID`, `ID_EXE`, `EXE_MEM`, `MEM_WB`) 来实现指令的并行执行。

#### 冒险解决方案

* **数据冒险：** 采用**数据前推（转发）+ 暂停（Stall）** 的方法。
    * **转发机制：** 在 EXE 级通过多路选择器，将 MEM 阶段 (`MEM_Result`) 或 WB 阶段 (`WB_BusW`) 的结果直接前推给 ALU 作为操作数，避免了大部分的数据相关停顿。
    * **暂停机制（Stall）：** 专门用于处理 `lw` 指令引发的 load-use 数据冒险。当检测到后续指令需要使用 `lw` 指令加载的数据时，生成 stall 信号，暂停 IF 和 ID 阶段的寄存器更新，从而在流水线中插入一个“气泡”（Bubble）等待数据准备就绪。
* **控制冒险：**
    * 将分支指令 (`beq`) 的条件判断 (`Branch` 信号) 和目标地址 (`bpc`) 的计算调整到 **EXE 级** 进行。
    * 将跳转指令 (`j`) 的目标地址 (`jpc`) 计算调整到 **ID 级** 进行。
    * 如果分支成立，系统会生成控制信号，废除（Flush）其后已经进入 IF 级和 ID 级的两条无效指令，替换为 `NOP` (空指令)。
    * 如果发生跳转，则废除其后已经进入 IF 级的指令。

## 核心模块说明

* `Fetch`: 整合了程序计数器 (PC)、指令存储器 (Inst_Rom)、地址加法器、数据扩展器 (EXT) 和多路选择器，完成取指操作。
* `Inst_Rom`: 指令存储器，预先加载了用于测试的机器码。
* `Registers`: 32 x 32 位的寄存器组，支持同时读取两个操作数并写入一个结果。
* `ALU`: 算术逻辑单元，负责执行算术和逻辑运算。
* `DataMem`: 数据存储器，用于 `lw` 和 `sw` 指令的访存操作。
* `CU`: 控制单元，根据指令的操作码生成各部件的控制信号。
* `IF_ID`, `ID_EXE`, `EXE_MEM`, `MEM_WB`: 各阶段间的流水线寄存器，负责在时钟上升沿传递数据和控制信号。
* `FPCPU`: 五级流水线顶层模块，实例化并连接了所有流水线阶段和冒险控制逻辑。

## 仿真与测试

项目包含了针对单周期 CPU 和流水线 CPU（包括冒险处理）的测试代码和预设指令序列。

1.  在 ISE 中打开工程。
2.  确认工程管理区中 `View` 选项设置为 `Simulation`。
3.  选中相应的测试模块（如顶层模块 `FPCPU` 的测试代码）。
4.  在过程管理区双击 `Simulate Behavioral Model` 运行仿真。
5.  通过观察波形图中的 `PC`, `IF_Inst`, `EXE_Result` 等核心信号，验证指令执行和冒险处理（如前推发生时的数据变化、Stall 发生时的 PC 停顿、分支废除时的空指令插入）是否正确。
